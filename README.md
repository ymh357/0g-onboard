# 0G Onboard

这是我整理的一套 0G 学习教程，从初学者到深度技术细节都有覆盖。

## 为什么做这个项目

在学习 0G 的过程中，我发现虽然白皮书和文档很详细，但缺少一个循序渐进的学习路径。很多初学者不知道从哪里开始，技术文档又太过晦涩。所以我想做一个交互式的教程平台，让学习 0G 变得更简单、更直观。

## 学习路径设计

我设计了三条学习路径，每条路径都有明确的目标受众和学习目标：

### 1. AI × Crypto 教程（基础概念）

**适合人群**：完全不了解 AI 和区块链交叉领域的新手

这条路径主要讲解 AI 和加密货币是如何结合的。我从最基础的概念开始，逐步引入去中心化 AI、智能合约、数据所有权等核心概念。内容来源于"AI x Crypto Primer"，但我加入了更多可视化图表和交互演示。

**核心问题**：
- 为什么 AI 需要区块链？
- 去中心化 AI 能解决什么问题？
- Web3 + AI 的未来是什么样的？

### 2. 0G 基础教程（入门 0G）

**适合人群**：了解区块链基础，想快速了解 0G 是什么

这条路径聚焦 0G 本身。我把白皮书的内容拆解成更容易理解的章节，每章都有明确的主题：
- 0G 是什么，要解决什么问题
- 核心架构（DA 层 + Storage 层 + Compute 层）
- 每一层的作用和工作原理
- 激励机制如何运作

**关键点**：不要死扣细节，先建立整体认知。

### 3. 0G 深度解析（技术深挖）

**适合人群**：开发者、技术研究者、想深入理解 0G 实现细节的人

这是最硬核的部分。每一章都深入一个技术主题，包含大量代码示例、架构图、工作流程演示。我花了很多时间整理这些内容，力求做到：
- 技术准确（基于白皮书和官方文档）
- 逻辑清晰（从"是什么"→"为什么"→"怎么做"）
- 实用性强（代码示例可直接参考）

**深度章节列表**：
1. **0G Chain 与多网络共识** - BFT 共识、多网络架构、分区设计
2. **Storage 层** - 存储节点、数据分片、冗余策略
3. **PoRA 证明机制** - 随机抽样、Merkle Proof、激励计算
4. **DA 层** - 数据可用性问题、Quorum 架构、证明流程
5. **Compute 层** - AI 模型推理、TEE 可验证性、训练服务
6. **INFT（智能 NFT）** - ERC-7857 标准、多模态、AI 生成
7. **Shared Staking** - 跨网络质押、安全性共享
8. **数据流** - 从上传到检索的完整流程
9. **对比分析** - 0G vs Celestia vs EigenDA
10. **实践指南** - 如何构建 0G 应用

## 内容组织逻辑

我在写每一章时都遵循一个固定的结构：

```
【是什么】→ 先给定义，建立概念
【为什么】→ 解释为什么需要这个设计
【怎么做】→ 深入技术实现细节
【应用】→ 实际场景和代码示例
```

这样的好处是，读者可以根据自己的需求选择阅读深度：
- 快速了解？只看"是什么"和"为什么"
- 深入研究？仔细阅读"怎么做"和代码
- 实践应用？直接看"应用"部分

## 技术选型

平台本身用 Next.js 14+ 构建，主要考虑：
- **静态生成**：教程内容不需要实时更新，静态生成速度最快
- **Markdown 驱动**：内容和代码分离，方便维护和扩展
- **交互式组件**：用 React + Framer Motion 做动画演示
- **响应式设计**：移动端和桌面端都能舒适阅读

核心依赖：
- Next.js 16（App Router）
- TypeScript 5
- Tailwind CSS 3
- Framer Motion 11（动画）
- Recharts 2（图表）
- Remark（Markdown 处理）

## 快速开始

```bash
# 安装依赖
bun install

# 启动开发服务器
bun dev

# 构建生产版本
bun run build
```

访问 http://localhost:3000 查看效果。

## 项目结构

```
0g-onboard/
├── content/chapters/        # 所有教程的 Markdown 文件
│   ├── 0*-*.md             # AI × Crypto 章节
│   ├── 0g-*.md             # 0G 基础章节
│   └── 0g-deep-*.md        # 0G 深度章节
├── content/data/           # 章节元数据（JSON）
├── components/             # React 组件
│   ├── ui/                # 基础 UI 组件（按钮、卡片等）
│   ├── tutorial/          # 教程组件（侧边栏、进度条）
│   └── interactive/       # 交互式图表和动画
├── app/                    # Next.js 页面路由
│   ├── tutorial/          # AI × Crypto 路由
│   ├── tutorial/0g/       # 0G 基础路由
│   └── tutorial/0g-deep-dive/  # 0G 深度路由
└── lib/                    # 工具函数（内容加载、格式化等）
```

## 如何添加新章节

1. 在 `content/chapters/` 创建 Markdown 文件
2. 在 `content/data/` 对应的 JSON 文件中添加元数据
3. 如果需要交互组件，在 `components/interactive/` 创建组件
4. 在章节页面中条件渲染组件

详细步骤看代码注释，或者参考现有章节的写法。

## 设计原则

**内容第一**：所有技术选型都服务于内容呈现。如果一个交互组件不能帮助理解，我就不加。

**渐进式学习**：从简单到复杂，从概念到实现，从理论到实践。读者可以在任何阶段停下来，不会有知识断层。

**保持更新**：0G 还在快速发展中，我会持续更新内容，确保准确性。
